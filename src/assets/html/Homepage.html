Auth API

Production-ready authentication backend with JWT rotation, Argon2 hashing, and full TypeScript strict mode. Built for the @vivekcsein Next.js frontend.

Hono Bun TypeScript strict Drizzle ORM PostgreSQL Argon2 passwords jose JWT Zod validation

Endpoints

POST /api/auth/signup Create account · sets cookies
POST /api/auth/signin Login · sets cookies
POST /api/auth/signout 🔒 auth required
POST /api/auth/refresh Silent token rotation
GET /api/auth/me 🔒 auth required
GET /health Liveness check
GET /static/* Binary assets (images, icons)
GET /views/* HTML templates + CSS/JS

Architecture

🔐
JWT Rotation
Access token (15 min) + refresh token (365 days). Both as httpOnly cookies. Silent refresh on 401 — user never re-logs.
🛡️
Argon2id Hashing
Passwords hashed with @node-rs/argon2 (Bun-native bindings). Timing-safe comparison. Same error for wrong email or wrong password.
🗄️
Session Store
V1: in-memory Map (zero deps, instant start). V2: Drizzle + Postgres sessions table. Same service API — swap the repository.
✅
Zod Everywhere
Every process.env var, every request body, every DB row validated before use. No any — unknown + narrowing only.
🚦
Rate Limiting
10 req / 15 min per IP on /signup and /signin. In-memory Map — no Redis needed for V1.
📁
File Serving
Two roots: /static for binary assets (30-day cache), /views for HTML templates (5-min cache in prod).

Quick Start

bash
# 1. Clone and install
git clone https://github.com/vivekcsein/hono-vercel-api
bun install

# 2. Configure environment
cp .env.example .env.local
# → fill in DATABASE_URL, JWT secrets

# 3. Run database migrations
bun run db:generate
bun run db:migrate

# 4. Start the dev server
bun run dev

# 5. Run tests
bun test
